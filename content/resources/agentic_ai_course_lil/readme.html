무엇을 구축할 것인가

V1: 작업 자율성(라우터 에이전트)

다음을 수행하는 고객 지원 라우팅 에이전트:

  • 고객 메시지를 적절한 부서로 분류합니다.
  • 기준 성능을 측정하기 위해 체계적인 평가를 사용합니다.
  • 개선 기회를 발견하기 위해 오류 분석을 적용합니다.
  • 목표한 개선을 구현하고 결과를 검증합니다.

주요 개념:

  • 간단한 프롬프트 엔지니어링
  • 체계적인 평가
  • 지속적 교정(CC): 실패를 분석하여 개선점 발견
  • 지속적 배포(CD): 개선 사항 구현 및 검증

V2: 계획 자율성(계획 대리인)

다음을 수행하는 다단계 계획 에이전트:

  • V1의 입증된 라우팅을 사용하여 메시지를 라우팅합니다(V1을 기반으로 구축!)
  • BM25를 사용하여 지식 베이스에서 관련 절차를 검색합니다.
  • 상세한 다단계 실행 계획을 생성합니다.
  • 사용자 정의 지표를 사용하여 검색 및 계획 품질을 측정하고 개선합니다.

주요 개념:

  • BM25를 사용한 RAG(검색 증강 생성)
  • 관찰을 통한 맞춤형 지표 설계
  • 계획 평가를 위한 LLM 판사
  • 증분 구축(V2 = V1 + 새로운 기능)

코스 구조

이 과정은 두 가지 자율성 수준에 대한 체계적인 패턴을 따릅니다.

  1. 빌드: 기준 시스템 구현
  2. 테스트: 평가를 실행하여 기준 측정항목 설정
  3. 보정(CC): 실패 관찰, 패턴 분석, 지표 설계
  4. 배포(CD): 목표한 개선, 재평가, 이익 검증

이 CC/CD 패턴은 프로덕션 시스템에 적합하며 모든 에이전트 AI 시스템을 체계적으로 개선하는 방법을 알려줍니다.
