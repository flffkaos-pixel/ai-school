응용 AI 현황 2025 보고서

2025년은 응용 AI에 있어서 획기적인 해였습니다.

이 보고서는 이러한 시스템을 제공하는 입력부터 생성되는 출력까지 전체 AI 애플리케이션 스택에 걸친 주요 개발 사항과 그 과정에서 직면하게 되는 과제를 정리합니다.

다루는 내용

1부: 입력 레이어

  • 프롬프트 엔지니어링에서 컨텍스트 엔지니어링으로: 모델의 취약성이 줄어들어 개별 프롬프트 작성에서 전체 컨텍스트 시스템 엔지니어링으로 초점이 이동했습니다.
  • 메타 프롬프트: AI 모델을 사용하여 최적화된 프롬프트 생성
  • 자동 프롬프트 최적화: 데이터 기반 프롬프트 개선을 위한 DSPy와 같은 프레임워크
  • 다중 모드 기본값: 텍스트 전용 AI 시스템은 이제 레거시입니다.

2부: 모델 계층

  • 추론 모델: o1 및 DeepSeek R1과 같은 모델을 사용하여 신뢰성을 위한 거래 속도
  • 장기 컨텍스트 창: 4K에서 100만 개 이상의 토큰으로 애플리케이션 구축 방법 변경
  • 효율적인 추론: 프로덕션 배포를 위한 양자화 및 최적화 기술

3부: 애플리케이션 계층

  • RAG 진화: 기본 검색부터 GraphRAG, 에이전트 검색 및 다중 모드 파싱까지
  • 프로덕션 에이전트: T자형 성공?봟간단한 작업의 도로 자동화, 특정 업종에 큰 영향
  • 코딩 에이전트: Cursor, Windsurf 및 Claude Code와 같은 도구를 갖춘 가장 성공적인 에이전트 카테고리
  • Human-in-the-Loop: 자동화와 적절한 인간 감독의 균형 유지

4부: 출력 레이어

  • 모델 및 제품 평가: 측정 대상과 이유 이해
  • 세 가지 평가 접근 방식: 주장, LLM 판사, 인간 평가
  • 지속적인 개선 플라이휠: 시간이 지남에 따라 개선되는 시스템 구축

5부: 과제

  • 환각: 이전보다 감지하기가 더 미묘하고 어렵습니다.
  • 일관되지 않은 추론: 이전에 해결한 문제에 대해 때때로 실패하는 모델
  • 과도한 자율성: 적절한 확인 없이 조치를 취하는 에이전트
  • 도구 호출 문제: 에이전트 시스템의 통합 문제

6부: 앞으로 나아갈 길

  • 지루한 인프라의 승리: 가장 효과적인 AI 작업은 데이터 및 아키텍처 기본 사항에 중점을 둡니다.
  • 통합 품질이 모델 선택보다 중요합니다: 어떤 모델을 선택하느냐보다 통합 방법이 더 중요합니다.
  • 평가는 협상 불가능합니다: 가치를 측정할 수 있는 팀은 계속 투자합니다.

주요 시사점

  1. 컨텍스트 엔지니어링 > 프롬프트 트릭 ??초기 프롬프트뿐만 아니라 전체 정보 환경을 설계합니다.
  2. 추론 모델은 신뢰성을 위해 속도를 교환합니다 ??정확성이 중요한 복잡한 작업에 사용합니다.
  3. RAG는 진화했지만 죽지 않았습니다 ??구조, 대행사 및 다중 모드 구문 분석으로 검색이 더욱 강력해졌습니다.
  4. 에이전트가 T자형 성공을 거두었습니다 ??간단한 작업의 광범위한 자동화, 특정 업종에 큰 영향
  5. 새로운 기능으로 인해 새로운 실패 모드가 발생함 ??미묘한 환각, 일관성 없는 추론, 과도한 자율성
  6. 평가는 병목현상이다 ??측정할 수 없는 것은 개선할 수 없다
  7. 인프라가 영웅적인 시스템을 이긴다 ??지루하고 안정적인 시스템이 영리하고 취약한 시스템보다 성능이 뛰어납니다.
