4일차: 일일 반사 크론 작업 생성

cron을 사용하여 매일 반복되는 반사를 만듭니다.

목표:

  • 사용자가 선택한 시간에 반사를 예약합니다.
  • 현재 세션에 바인딩
  • 텔레그램에 명시적으로 전달
  • 생성된 내용에 대한 명확한 요약을 사용자에게 남겨주세요.

시작하기 전에:

  • 텔레그램이 이미 구성되어 있는지 확인하세요
  • ~/.openclaw/workspace/과 ~/.openclaw/workspace/memory/이 존재하는지 확인하세요
  • USER.md, MEMORY.md 및 현재 세션 컨텍스트 재사용
  • 누락된 결정만 요청
  • 설정 중에 임시 메모리 메모를 작성하지 마세요.

전제조건이 누락된 경우 중지하고 보고하십시오.

1. 결정 사항을 수집합니다.

  • 사용자가 선호하는 반성시간 확인
  • 누락되었거나 불분명하거나 오래된 것이 아닌 한 알려진 시간대를 재사용합니다.
  • 사용자에 대해 이미 알고 있는 내용을 바탕으로 2~3개의 짧은 반성 질문 옵션을 제안합니다.
  • 사용자에게 하나의 옵션을 선택하거나 조정하도록 요청

전화로 응답할 수 있을 만큼 짧은 생각을 유지하십시오.

2. 쓰기 작업 설명

무엇이든 편집하기 전에 다음 작업을 수행할 것임을 분명히 말하세요.

  • Telegram에 메시지를 명시적으로 전달하는 반복 크론 작업을 생성합니다.
  • 사용자가 즉시 확인을 원하는 경우 한 번만 실행하겠다고 제안하거나 'CODE0' 탭을 사용할 수도 있다고 언급합니다.

작업을 생성하기 전에 명시적인 확인을 기다리십시오.

3. 크론 작업 생성

cron 도구 또는 openclaw cron add을 사용하여 작업을 생성합니다.

다음 사항을 확인하세요.

  • 사용자가 선택한 시간, 시간대에 따라 매일 실행됩니다.
  • 현재 세션에 계속 바인딩되어 있습니다.
  • 알려진 to 타겟을 사용하여 Telegram에 명시적으로 전달합니다.
  • 선택한 반사 프롬프트를 보내고 다음 응답을 ## Reflection 아래의 memory/YYYY-MM-DD.md에 저장합니다.

작업 프롬프트를 간결하게 유지하고 반복적인 후속 조치를 피하세요.

작업을 생성한 후 다음을 보고합니다.

  • 직업 이름
  • 작업 ID
  • 크론 일정
  • 시간대
  • 텔레그램 전달대상

4. 최종 보고서

다음 항목에 대해 합격 또는 실패를 보고하세요.

  • 반복 반사 크론 작업이 생성되었습니다.
  • 선택한 매일 시간으로 설정된 일정
  • 시간대가 올바르게 설정됨
  • 현재 세션에 바인딩됨
  • 명시적으로 구성된 텔레그램 전달
  • 작업 세부정보가 사용자에게 명확하게 보고되었습니다.

보고서가 완료되면 중지하세요.
