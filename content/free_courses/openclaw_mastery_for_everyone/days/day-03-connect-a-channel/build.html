3일차 빌드: 채널 연결

이것은 3일차에 대한 사용자용 가이드입니다. 오늘 Telegram을 연결하여 휴대폰에서 Claw와 대화할 수 있습니다.

운영 단계는 claw-instructions-connect-telegram.md에 있습니다.

시작하기 전에 필요한 것

  • 1일차 완료: OpenClaw 설치 및 작동
  • 2일차 완료: ID 파일이 생성되고 올바르게 로드됨
  • 휴대폰이나 데스크탑에 설치된 텔레그램
  • 웹 채팅을 통해 Claw에 접근

이 단계의 모든 구성은 ~/.openclaw/openclaw.json에 있습니다.

1단계: 웹 채팅에서 설정 시작

다음 메시지를 복사하여 웹 채팅에 붙여넣습니다.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-03-connect-a-channel/claw-instructions-connect-telegram.md을 이해하고 모든 단계를 추가하세요.

claw-instructions-connect-telegram.md은 Claw에게 다음을 지시합니다.

  • 텔레그램 봇 토큰과 사용자 ID를 한 번에 하나씩 수집하세요.
  • ~/.openclaw/openclaw.json 업데이트
  • 기본적으로 텔레그램을 비공개로 유지
  • 게이트웨이 다시 시작
  • 휴대폰에서 페어링 및 실시간 테스트 방법을 안내합니다.

높은 수준에서 수행하는 작업은 다음과 같습니다.

  • Telegram에서 BotFather를 사용하여 봇을 만들고 토큰을 얻습니다.
  • Claw가 요청할 때 해당 토큰을 Claw에 제공하세요.
  • Claw가 페어링 과정을 안내하는 동안 Telegram의 지침을 따르세요.
  • 불분명한 점이 있으면 이동하는 동안 웹 채팅에서 Claw에 대해 질문하세요.

설정이 완료되면 Telegram에서 봇에게 문자 메시지를 보내고 응답을 받을 수 있습니다.

발톱이 당신에게 요구해야 할 것

설정 중에 Claw가 다음을 요청할 것으로 예상됩니다.

  • BotFather의 텔레그램 봇 토큰
  • 숫자로 된 텔레그램 사용자 ID

그런 다음 ~/.openclaw/openclaw.json의 channels.telegram 아래에 Telegram을 구성하고 채널을 제한적으로 유지해야 합니다.

  • dmPolicy: "pairing"
  • groupPolicy: "disabled"

텔레그램을 plugins.entries.telegram 아래에 배치하면 잘못된 것입니다.

편성

Claw가 Telegram을 구성하고 게이트웨이를 다시 시작하면 다음이 수행됩니다.

  1. 텔레그램에서 봇 찾기
  2. /start 보내기
  3. 페어링 요청 승인
  4. 휴대폰에서 실제 메시지 보내기

작동하면 채널이 활성화됩니다.

검증하다

이는 빠른 승리보다 더 간단해야 합니다.

문제 해결


Claw에서는 Telegram이 플러그인이라고 말합니다
그렇지 않습니다. 


설정 후 봇이 응답하지 않습니다.
Claw에게 게이트웨이 로그를 확인하고 봇 토큰이 누락된 문자나 추가 공백 없이 올바르게 복사되었는지 확인하도록 합니다.


페어링이 작동하지 않습니다
숫자 텔레그램 사용자 ID가 allowFrom에 있고 따옴표 붙은 문자열이 아닌 숫자로 저장되어 있는지 확인하세요.


Claw는 텔레그램에서 응답하지만 일반적으로 들립니다.
작업 공간 경로가 여전히 올바른지, 2일차의 ID 파일이 로드되는지 확인하세요.


토큰은 유효한 것처럼 보이지만 Telegram은 여전히 ​​실패합니다.
Claw에게 콜론 앞의 숫자 접두사를 포함하여 전체 BotFather 토큰이 한 줄에 복사되었는지 확인하도록 요청하세요.
