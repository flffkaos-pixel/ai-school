빌드 1일차: OpenClaw 설치 및 보안

이것은 1일 차 빌드 가이드입니다. 전반부에서는 Hostinger에 OpenClaw를 배포하는 과정을 안내합니다.

1단계: Hostinger에 배포

Hostinger 계정을 만들고 VPS에서 OpenClaw를 시작하세요

LLM 중 하나에 대한 API 키가 아직 없는 경우 이 가이드에 따라 API 키를 받으세요.

여기를 클릭하고 아래 동영상을 따라 Hostinger 계정을 만들고, VPS를 만들고, OpenClaw를 실행하세요.

채팅창에 응답이 표시되면 Claw가 활성화된 것입니다.

2단계: 보안 확인

이제부터 모든 것은 Claw와의 대화입니다.

4단계: 보안 확인 실행

다음 메시지를 복사하여 웹 채팅에 붙여넣습니다.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-01-install-secure/claw-instructions-security.md을 이해하고 모든 단계를 추가하세요.

claw-instructions-security.md에는 OS, 개방형 포트, 방화벽, OpenClaw의 보안 감사, 게이트웨이 구성, 파일 권한, 채널,

실행되는 동안 채팅에 이와 같은 진행 블록이 표시됩니다.

완료되면 각 항목이 PASS, FAIL 또는 EXPECTED로 표시된 요약이 표시됩니다.

claw-instructions-security.md을 직접 열어 자신의 속도에 맞춰 점검 사항과 예상 결과를 읽어보실 수도 있습니다.

5단계: 클로 이름 지정

귀하의 Claw는 귀하의 이름을 묻고 이름을 지정하도록 요청할 것입니다.

또한 몇 가지 추가 설정 질문을 물을 수도 있습니다.

한 번의 빠른 승리

감사가 완료되면 다음을 시도해 보십시오.

내 설정을 간단하게 영어로 설명해주세요.

이렇게 하면 설정을 읽기 쉽게 만듭니다.

1일차 이후에 사실이어야 하는 사항

  • Claw가 웹 채팅에서 응답합니다.
  • openclaw security audit에는 심각한 오류가 표시되지 않습니다.
  • 토큰 인증이 활성화된 127.0.0.1에 바인딩된 게이트웨이
  • DM 및 그룹 정책은 제한적입니다(명시적으로 설정하거나 안전한 기본값을 사용함).
  • ~/.openclaw/credentials에는 700 권한이 있습니다.
  • 하트비트가 0m으로 설정됨
  • 아직 자격 증명이 저장된 채널이 없습니다.
  • 웹 검색이 비활성화되었습니다.
  • 발톱에 이름이 있습니다

이 모든 사항이 충족되면 Day 1이 완료됩니다.

문제 해결

Claw가 웹 채팅에서 응답하지 않습니다.
게이트웨이를 다시 시작해야 할 수도 있습니다.

Claw가 문제를 해결하려고 시도한 후 보안 감사에 실패가 표시됩니다.
일부 수정 사항은 적용되기 전에 게이트웨이를 다시 시작해야 합니다.

Claw는 일반적인 응답을 제공하고 검사를 실행하지 않습니다.
claw-instructions-security.md에 액세스하지 못할 수도 있습니다.

방화벽이 FAIL로 표시됩니다.
이는 Docker 컨테이너 내부에서 예상되는 현상입니다.
