2일차: ID 설정 마무리

다음 지침을 정확하게 따르십시오.

어떤 작업을 하기 전에 다음 파일이 있는지 확인하세요.

  • ~/.openclaw/workspace/SOUL.md
  • ~/.openclaw/workspace/USER.md
  • ~/.openclaw/workspace/AGENTS.md
  • ~/.openclaw/workspace/MEMORY.md
  • ~/.openclaw/workspace/memory/

누락된 것이 있으면 중지하고 누락된 내용을 보고하십시오.

1. 파일 권한 설정

다음 권한을 설정하십시오.

  • SOUL.md: 600
  • USER.md: 600
  • AGENTS.md: 600
  • MEMORY.md: 600
  • memory/ 디렉토리: 700

권한을 변경한 후 다시 읽고 결과를 보고하세요.

2. 게이트웨이 다시 시작

달리다:

openclaw 게이트웨이 다시 시작





3. 로드된 ID 확인


활성 OpenClaw 인스턴스를 통해 두 확인 프롬프트를 모두 실행합니다.


테스트 1


당신은 나에 대해 무엇을 알고 있나요?



테스트 2


당신의 규칙은 무엇입니까?



응답이 일반적인 경우 작업 공간 파일이 로드되지 않았을 수 있다고 보고하고 openclaw.json에 구성된 작업 공간 경로를 확인하라고 말합니다.
