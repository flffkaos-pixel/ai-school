9일차 빌드: 팀 제공

이것은 9일차에 대한 사용자용 가이드입니다. 전체 강의 동안 OpenClaw 채팅에 참여하세요.

시작하기 전에 필요한 것

  • 1일차 완료: OpenClaw 설치 및 보안
  • 2일차 완료: ID 파일이 생성되고 올바르게 로드됨
  • 3일차 완료: 텔레그램 연결 및 작동 중
  • 4일차 완료: 사전 예방적 워크플로가 이미 존재합니다.
  • 5일차 완료: 기술이 작동 중입니다.
  • 6일차 완료: 이메일 분류가 작동 중입니다.
  • 7일차 완료: 웹 검색이 작동 중입니다.
  • 8일차 완료: 승인이 포함된 아웃바운드 이메일이 작동 중입니다.
  • 웹 채팅을 통해 OpenClaw에 접근

9일차를 달리는 방법

다음 순서대로 단계를 진행하세요.

  1. 채팅에서 작가 모델을 확인하세요
  2. claw-instructions-create-writer-agent.md
  3. claw-instructions-enable-teamwork.md
  4. 작성자 및 위임 확인 실행

이 순서는 설정을 명확하게 유지합니다.

1단계: 작성자 모델 확인

다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

무엇이든 하기 변경하기 때문에 현재 설정을 검사하고 기본 Claw가 어떤 기본 모델을 사용하고 있는지, 작성자가 gpt-5.4, claude-sonnet-4.6 또는 Gemini 3 Flash를 포함한다고 합니다.

이는 새 에이전트가 존재하기 전에 모델 결정을 명시적으로 만듭니다.

2단계: writer 생성

계획이 만족스러우면 다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-09-give-it-a-team/claw-instructions-create-writer-agent.md을 이해하고 모든 단계를 추가하세요.

claw-instructions-create-writer-agent.md은 Claw에게 다음을 지시합니다.

  • 현재 구성 및 기본 ID 파일을 먼저 검사하십시오.
  • 주제, 청중, 목소리에 관해 몇 가지 짧은 질문을 해보세요.
  • 자체 작업 공간을 사용하여 writer이라는 이름의 에이전트를 만듭니다.
  • 자세한 SOUL.md과 범위가 지정된 USER.md, AGENTS.md 및 시작 MEMORY.md을 작성하세요.
  • 기본 설정에서 이미 실행 중인 모델 계열을 기반으로 gpt-5.4, claude-sonnet-4.6 또는 Gemini 3 Flash를 선택합니다.

작가 'CODE0'의 디테일이 포인트다.

3단계: 팀 연결

writer이 생성된 후 이를 복사하여 웹 채팅에 붙여넣습니다.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-09-give-it-a-team/claw-instructions-enable-teamwork.md을 이해하고 모든 단계를 추가하세요.

claw-instructions-enable-teamwork.md은 Claw에게 다음을 지시합니다.

  • main과 writer 사이에서만 에이전트 간 통신을 활성화합니다.
  • 기본 작업 공간 `CODE0`에 짧은 위임 규칙 하나를 추가하세요.
  • 무엇이 변경되었는지, 무엇이든 다시 로드해야 했는지 정확하게 보고하세요.

이를 통해 책임이 명확해집니다.

  • 다시 main을 클릭하세요

9일차 이후에는 무엇이 사실이어야 할까요?

지정 대리인

  • writer이라는 에이전트가 존재합니다.
  • 작성자는 기본 설정과 동일한 공급자 제품군에서 선택한 gpt-5.4, claude-sonnet-4.6 또는 Gemini 3 Flash를 사용합니다.
  • 작성기 작업공간에는 긴 형식의 쓰기에 맞게 조정된 자세한 SOUL.md이 있습니다.
  • 작성기 작업공간에는 USER.md, AGENTS.md, MEMORY.md가 있습니다.
  • main과 writer 사이에 에이전트 간 통신이 활성화되어 있습니다.
  • 기본 작업공간에는 장문 쓰기에 대한 짧은 위임 규칙이 있습니다.
  • 작성자 전용 테스트를 통해 설득력 있는 초안이 생성되었습니다.
  • 위임된 초안이 기본 Claw를 통해 돌아왔습니다.
  • 수정 요청이 작성자를 통해 왕복되었습니다.

문제 해결

메인 Claw는 초안 자체를 작성합니다.
보다 직접적으로 질문하십시오: Use the writer agent for this draft. 여전히 해당 부분 자체를 작성하는 경우, 메인 작업공간 AGENTS.md에 추가한 긴 형식 위임 규칙을 보여달라고 요청하십시오.

작가가 일반적으로 들립니다.
Claw에게 작가 SOUL.md을 보여달라고 요청하고 음성 섹션을 조여보세요.

작성기 에이전트가 아직 인터페이스에 나타나지 않습니다.
9일차 설정이 완전히 다시 로드되었는지, 작성기 작업공간이 생성된 위치가 무엇인지 기본 Claw에 문의하세요.

위임 실패
Claw에 현재 에이전트 간 허용 목록을 검사하고 main에서 writer 연결만 수정하도록 요청하세요.

작성기 실행 후 채팅에서 후속 메시지 수신이 중지됩니다.
채팅 창 상단에 있는 세션 전환기를 사용하세요.
