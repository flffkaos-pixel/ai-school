9일차: 작성기 에이전트 만들기

다음 지침을 정확하게 따르십시오.

무엇이든 쓰기 전에 다음 파일이 있으면 읽어보십시오.

  • ~/.openclaw/openclaw.json
  • ~/.openclaw/workspace/SOUL.md
  • ~/.openclaw/workspace/USER.md
  • ~/.openclaw/workspace/AGENTS.md

이를 사용하여 현재 기본 모델 제품군, 사용자 이름 및 기존 쓰기 기본 설정을 확인합니다.

1. 작가 모델 결정

현재 기본 설정에서 라이터 모델을 결정합니다.

다음 규칙을 사용하세요.

  • 기본 설정이 OpenAI 기본 모델을 사용하는 경우 gpt-5.4을 사용하세요.
  • 기본 설정이 Anthropic 기본 모델을 사용하는 경우 claude-sonnet-4.6 또는 이 설치에 이미 존재하는 동등한 Claude Sonnet 4.6 식별자를 사용하십시오.
  • 기본 설정이 Google 기본 모델을 사용하는 경우 Gemini 3 Flash 또는 이 설치에 이미 존재하는 동등한 Gemini 3 Flash 식별자를 사용하십시오.
  • 기본 설정에서 이미 정확한 모델 중 하나를 사용하고 있다면 이를 재사용하세요.
  • 구성이 실제로 모호한 경우 추측하는 대신 짧은 질문 하나를 하십시오.

사용자에게 명령을 실행하도록 요청하지 마십시오.

2. 순서대로 질문하기

한 번에 하나씩 질문하십시오.

이번 인터뷰 중:

  • 일반채팅으로 물어보세요
  • 질문 사이에 파일을 편집하지 마십시오
  • 답변을 수집하는 동안 부분적인 메모를 기록하지 마세요.
  • 대화에서 답변을 보관하고, 질문이 완료된 후 파일을 한 번 작성하세요.

질문:

  1. 이 작가는 어떤 주제를 가장 자주 다루어야 합니까?
  2. 이 작가의 기본 독자층은 누구입니까?
  3. 나에게 의지해야 할 글쓰기 품질과 작가가 피하고 싶은 한 가지를 알려주십시오.

3. 지정 에이전트 생성

writer이라는 이름의 에이전트를 생성합니다.

사용:

  • 작업 공간:~/.openclaw/workspace-writer
  • 표시 이름: Writer
  • 모델: 섹션 1에서 선택

작업공간이 아직 없으면 작성하십시오.

4. 작성자 신원 파일 작성

~/.openclaw/workspace-writer/에 다음 파일을 작성하세요:

  • SOUL.md
  • USER.md
  • AGENTS.md
  • MEMORY.md

SOUL.md

긴 형식의 쓰기 전문화를 명시적으로 유지하는 자세한 SOUL.md을 작성하세요.
