6일 차 구축: 받은 편지함 관리하기

이것은 6일차에 대한 사용자용 가이드입니다. 오늘은 받은 편지함 흐름을 단계별로 수행합니다.

하루 종일 개인 Gmail받은 편지함을 가정합니다.

시작하기 전에 필요한 것

  • 1일차 완료: OpenClaw 설치 및 보안
  • 2일차 완료: ID 파일이 생성되고 올바르게 로드됨
  • 3일차 완료: 텔레그램 연결 및 작동 중
  • 4일차 완료: 사전 예방적 워크플로가 이미 존재합니다.
  • 5일차 완료: 이미 ClawHub 스킬을 한 번 검사하고 설치했습니다.
  • 웹 채팅을 통해 Claw에 접근
  • 개인 Gmail 계정에 액세스
  • 브라우저에서 Google 계정 설정을 여는 기능

6일차 달리기 방법

다음 순서로 파일을 살펴보세요.

  1. Gmail 앱 비밀번호 만들기
  2. 채팅에서 imap-smtp-email 검사
  3. claw-instructions-install-imap-smtp-email.md
  4. claw-instructions-create-email-triage.md
  5. claw-instructions-finalize-inbox.md

이 순서가 중요합니다.

이 날에는 4일차에 사용한 것과 동일한 크론 경로를 유지하세요. 이는 정확한 시간 아침 배송을 위해 Hostinger에 더 적합합니다.

1단계: Gmail 앱 비밀번호 만들기

앱 포스틱를 엽니다.

Google에서 먼저 다른 곳으로 보내는 경우 2단계인증을 사용 설정하고 앱 비밀번호 페이지로 돌아오세요.

새 앱 비밀번호를 만드세요:

  • 앱 이름:openclaw-imap
  • Google이 생성한 16자리 비밀번호를 복사하세요.

Google은 각 앱 비밀번호를 한 번씩 표시합니다.

2단계: imap-smtp-email 검사

다음 내용을 복사하여 OpenClaw 웹 채팅에 붙여넣으세요.

ClawHub에서 imap-smtp-email을 검사하고 그 기능, 필요한 Gmail 자격 증명, 구성 위치, 위험할 수 있는 항목, 해당 편지함 밖에만 6일차를 찾을 수 있는 방법을 일반 영어로 설명합니다.

여기서는 스킬이 이름과 일치하는지, 동작이 오늘의 경계에 맞는지 두 가지를 확인합니다.

3단계: imap-smtp-email 설치

검사가 만족스러우면 다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-06-tame-your-inbox/claw-instructions-install-imap-smtp-email.md을 이해하고 모든 단계를 추가하세요.

지침 파일은 Claw에게 다음을 지시합니다.

  • 이 작업 공간에 imap-smtp-email을 설치하세요
  • 필요한 경우 Gmail 주소와 앱 비밀번호를 물어보세요.
  • ~/.config/imap-smtp-email/.env에서 Gmail IMAP 측을 구성하십시오.
  • 8일차에는 SMTP 측을 떠나십시오.
  • 스킬과 구성이 어디에 있는지 알려주세요.

이 단계를 마친 후 계속 진행하기 전에 OpenClaw에 /new을 입력하세요.

4단계: email-triage 생성

다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-06-tame-your-inbox/claw-instructions-create-email-triage.md을 이해하고 모든 단계를 추가하세요.

지침 파일은 Claw에게 다음을 지시합니다.

  • email-triage 작업 공간 스킬 생성
  • 특정 이메일을 요청하지 않는 한 보낸 사람, 제목, 카테고리 및 개수에 대한 요약을 유지합니다.
  • AGENTS.md에 Email Security Protocols을 추가하세요.
  • 아침에 반복되는 Gmail 크론 작업 만들기

받은 편지함을 자신의 편지함처럼 느끼게 하는 레이어입니다.

이 단계를 마친 후 계속 진행하기 전에 OpenClaw에 /new을 입력하세요.

5단계: 마무리 및 확인

다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-06-tame-your-inbox/claw-instructions-finalize-inbox.md을 이해하고 모든 단계를 추가하세요.

지침 파일은 다음을 알려줍니다.

  • Gmail IMAP 설정이 존재하는지 확인하세요
  • 6일차 받은 편지함 읽기에 'CODE0'이 사용 가능한지 확인하세요.
  • email-triage, AGENTS.md 및 아침 크론 작업이 있는지 확인하세요.
  • 확인 결과를 PASS 또는 FAIL로 보고합니다.

6일차 이후에는 무엇이 사실이어야 할까요?

  • A personal Gmail App Password was created
  • 설치 전 imap-smtp-email을 검사했습니다.
  • imap-smtp-email was installed from ClawHub for this workspace
  • Gmail IMAP settings were stored in ~/.config/imap-smtp-email/.env
  • ~/.config/imap-smtp-email/.env permissions are owner-only
  • SMTP settings are still left for Day 8
  • email-triage이 작업 공간 스킬로 존재합니다.
  • AGENTS.md에는 이메일 보안 프로토콜이 포함되어 있습니다.
  • 아침 Gmail 요약에 대한 반복 크론 작업이 있습니다.
  • 크론 작업 일정이 선택한 아침 시간과 일치합니다.
  • 크론 작업 시간대가 귀하의 시간대와 일치합니다.
  • Claw는 구조화된 Gmail 분류 요약을 반환할 수 있습니다.
  • Claw가 프롬프트 주입 텍스트를 따르는 대신 플래그를 지정합니다.

문제 해결

Google 계정에서 앱 비밀번호를 찾을 수 없습니다.
공식 Google 도움말 페이지(앱 포스트로 로그인)를 확인하세요.

Gmail에서 비밀번호가 잘못되었다고 합니다.
일반 Gmail 비밀번호가 아닌 16자리 앱 ​​비밀번호를 사용하세요.

스킬은 Gmail을 읽을 수 있지만 전송 측도 구성된 것처럼 보입니다.
6일차에는 SMTP가 범위를 벗어납니다.

요약에 이메일 본문 텍스트가 너무 많이 표시됩니다.
특정 이메일을 요청하지 않는 한 요약이 보낸 사람, 제목, 카테고리 및 개수에 머물도록 Claw에 'CODE0' 기능을 강화하도록 요청하세요.

아침 요약이 도착하지 않습니다
Claw에게 크론 작업 일정, 시간대, 세션 대상, 텔레그램 전달 대상을 함께 검사하도록 요청하세요.

중복된 아침 요약을 받기 시작합니다.
Claw에 활성 크론 작업 목록을 요청하고 비활성화하거나 제거해야 하는 오래된 아침 요약 작업을 찾으세요.

새 기술이 아직 활성화되지 않은 것 같습니다.
테스트하기 전에 OpenClaw에 /new을 입력하세요.
