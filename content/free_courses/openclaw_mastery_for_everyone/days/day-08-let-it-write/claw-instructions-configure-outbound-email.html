8일차: 아웃바운드 이메일 구성

목표: 이 작업 공간에 대해 imap-smtp-email의 Gmail SMTP 측을 활성화하고 8일차 아웃바운드 이메일 규칙을 추가합니다.

주요 제약사항:

  • 사용자에게 쉘 명령을 실행하도록 요청하지 마십시오.
  • 6일차 Gmail 설정이 이미 있는 경우 다시 사용하세요.
  • 작성 전용 이메일에 8일을 유지하세요.
  • 모든 아웃바운드 이메일은 전체 내용이 표시되어야 하며 전송하기 전에 명시적인 승인이 필요합니다.

하다:

  1. imap-smtp-email이 이미 설치되어 준비되어 있는지 확인하세요.
  2. 여전히 필요한 경우에만 Gmail 주소와 앱 비밀번호를 요청하세요.
  3. ~/.config/imap-smtp-email/.env:
      에서 Gmail SMTP 설정을 구성합니다.
    • SMTP_PORT=587
    • SMTP_USER은 사용자의 전체 Gmail 주소여야 합니다.
    • SMTP_PASS은 Gmail 앱 비밀번호여야 합니다.
    • SMTP_FROM은 Gmail 주소와 일치해야 합니다.
  4. 사용자에게 비밀번호를 다시 인쇄하지 않고 구성 파일 권한이 소유자 전용인지 확인하십시오.
  5. AGENTS.md에 다음과 같은 Outbound Email Protocols 섹션을 추가하세요:
    • To, Subject 및 Body가 포함된 전체 초안을 표시합니다.
    • 사용자가 다른 톤을 요구하지 않는 한 'CODE0'의 톤을 일치시킵니다.
    • 첨부 파일에는 명시적인 사용자 지침과 파일 확인이 필요합니다.
    • 받는 사람 주소가 확실하지 않으면 추측하지 말고 물어보세요.
    • 8일차 범위는 새 메시지 작성만입니다.
  6. 사용자에게 정확히 무엇이 변경되었는지 알려주세요.
  7. 사용자에게 다음에 실행할 확인 메시지를 하나 제공합니다.

최종 답변에는 다음이 포함됩니다.

  • 합격 또는 실패
  • imap-smtp-email이 준비되었는지 여부
  • 비밀을 인쇄하지 않고 SMTP 설정이 저장된 위치
  • AGENTS.md 업데이트 여부
  • 정확한 검증 프롬프트
