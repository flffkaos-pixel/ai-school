8일차: follow-up-email 생성

목표: 이 작업 공간에 대한 follow-up-email 스킬을 만듭니다.

주요 제약사항:

  • 8일차에는 작성 전용 경로에 스킬을 유지합니다.
  • 보내기 전에 항상 승인이 필요합니다.
  • 수신자 이메일 주소를 추측하지 마세요.
  • 이메일 본문은 직렬화된 문자열이 아닌 사람이 읽을 수 있는 일반 텍스트여야 합니다.

하다:

  1. 작업 공간 스킬로 follow-up-email을 생성합니다.
  2. send a follow-up to ... about ...과 같은 요청에 대해 트리거되도록 만듭니다.
  3. 다음을 사용하여 자세한 SKILL.md을 작성하세요:
    • 짧은 "무엇을 하는가" 섹션
    • 다음과 같은 워크플로:
      • 150단어 미만의 짧은 후속 이메일 초안을 작성합니다.
      • Follow-up: [topic]과 같은 제목을 사용합니다.
      • 명확한 다음 단계가 포함되어 있거나 질문하십시오.
      • 이메일 본문을 실제 단락 구분이 있는 실제 일반 텍스트로 렌더링합니다.
      • \n과 같은 이스케이프된 개행 시퀀스, 인용된 JSON 스타일 본문 문자열, 마크다운 코드 울타리 또는 이메일 본문의 길 잃은 접두사 문자를 출력하지 않습니다.
      • 보내기 전에 승인을 위해 전체 초안을 제시합니다.
    • 불확실한 수신자, 첨부 파일, 서식 실수 및 작성 전용 이메일 내부 유지에 대한 가드레일
  4. 기술 지침에서 승인 전 최종 자체 점검을 추가하세요.
    • 본문에 \n, 선행 따옴표, 주변 따옴표, 코드 펜스 또는 기타 직렬화 아티팩트와 같은 이스케이프 시퀀스가 ​​포함된 경우 사용자에게 표시하기 전에 깨끗한 일반 텍스트로 다시 작성하세요.
  5. 새로운 기술을 테스트하기 전에 OpenClaw에서 /new을 입력하도록 사용자에게 안내합니다.

최종 답변에는 다음이 포함됩니다.

  • 합격 또는 실패
  • 스킬이 생성된 곳
  • 테스트할 정확한 트리거 문구
  • 다음에 실행하라는 메시지 예시

거기 멈춰라.
