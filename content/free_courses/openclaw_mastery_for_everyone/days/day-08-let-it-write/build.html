8일차 빌드: 직접 작성해 보세요

이것은 8일차에 대한 사용자용 가이드입니다. 오늘은 6일차의 Gmail 기술로 돌아가서 보내기 측을 켜고 아웃바운드 이메일 규칙을 추가하고 그 위에 하나의 작은 후속 작업 흐름을 만듭니다.

운영 단계는 이 파일과 몇 개의 작은 지침 파일로 나누어져 있습니다.

시작하기 전에 필요한 것

  • 1일차 완료: OpenClaw 설치 및 보안
  • 2일차 완료: ID 파일이 생성되고 올바르게 로드됨
  • 3일차 완료: 텔레그램 연결 및 작동 중
  • 4일차 완료: 사전 예방적 워크플로가 이미 존재합니다.
  • 5일차 완료: 기술이 작동 중입니다.
  • 6일차 완료: Gmail 받은편지함 읽기가 'CODE0'을 통해 작동 중입니다.
  • 7일차 완료: 웹 검색이 작동 중입니다.
  • OpenClaw 웹 채팅을 통해 Claw에 접근하세요.
  • 6일차에 사용한 것과 동일한 개인 Gmail 계정에 액세스

8일차 달리기 방법

다음 순서대로 단계를 진행하세요.

  1. 채팅에서 imap-smtp-email의 전송측을 검사하세요
  2. claw-instructions-configure-outbound-email.md
  3. claw-instructions-create-follow-up-email.md
  4. claw-instructions-finalize-outbound-email.md

이 순서는 설정을 읽기 쉽게 유지합니다.

1단계: imap-smtp-email의 송신 측 검사

다음 내용을 복사하여 OpenClaw 웹 채팅에 붙여넣으세요.

ClawHub에서 'CODE0'을 다시 검사합니다.

여기서는 두 가지 사항, 즉 기술이 전송 워크플로에 준비되어 있는지 여부와 8일차 경계가 여전히 명확한지 여부를 확인합니다.

2단계: 아웃바운드 이메일 구성

검사가 만족스러우면 다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-08-let-it-write/claw-instructions-configure-outbound-email.md을 이해하고 모든 단계를 추가하세요.

지침 파일은 Claw에 다음을 지시합니다.

  • imap-smtp-email이 설치되어 있고 준비되었는지 확인하세요.
  • 6일차 Gmail 주소와 앱 비밀번호가 여전히 필요한 경우 다시 사용하세요.
  • Gmail SMTP 설정을 ~/.config/imap-smtp-email/.env에 추가하세요
  • 구성 파일을 소유자만 유지
  • AGENTS.md에 Outbound Email Protocols을 추가하세요.
  • 모든 전송 전에 명시적인 승인을 받아 작성 전용 이메일에 8일차 유지

6일차 앱 비밀번호는 여기에서 사용하는 것과 동일한 자격 증명입니다.

3단계: follow-up-email 생성

아웃바운드 이메일이 구성된 후 다음을 복사하여 웹 채팅에 붙여넣습니다.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-08-let-it-write/claw-instructions-create-follow-up-email.md을 이해하고 모든 단계를 추가하세요.

안내파일은 Claw에게 다음과 같은 사용자 정의 작업 공간 기술을 생성하라고 지시합니다.

  • send a follow-up to ... about ...과 같은 요청에 대해 트리거됩니다.
  • 수신자 주소를 찾거나 요청합니다.
  • Claw의 목소리로 짧은 후속 이메일을 작성합니다.
  • 실제 단락 구분을 사용하여 본문을 깨끗한 일반 텍스트로 형식화합니다.
  • 보내기 전에 승인을 위한 전체 초안을 보여줍니다.
  • 8일차 작성 전용 경계 내에 유지됩니다.

이 단계 후에 새로운 기술을 테스트하기 전에 OpenClaw에 /new을 입력하세요.

다음 내용을 복사하여 웹 채팅에 붙여넣으세요.

https://raw.githubusercontent.com/aishwaryanr/awesome-generative-ai-guide/main/free_courses/openclaw_mastery_for_everyone/days/day-08-let-it-write/claw-instructions-finalize-outbound-email.md을 이해하고 모든 단계를 추가하세요.

안내파일은 다음을 알려줍니다.

  • Gmail SMTP 설정이 존재하는지 확인하세요
  • imap-smtp-email이 설치되어 있고 준비되었는지 확인하세요.
  • Outbound Email Protocols 및 follow-up-email이 있는지 확인하세요.
  • 한 번의 정확한 승인 테스트와 한 번의 정확한 취소 테스트를 제공합니다.

문제 해결


Claw는 새 이메일을 작성하는 대신 답장이나 전달을 시작합니다.
8일차는 작성만 가능하다고 알려주세요. 


SMTP 인증 실패
6일차와 동일한 Gmail 앱 비밀번호를 사용하세요. Google이 이를 거부하면 새 앱 비밀번호를 생성하고 Claw가 ~/.config/imap-smtp-email/.env에서 IMAP 및 SMTP 값을 모두 업데이트하도록 하세요.


Claw에서 Gmail 전송 설정이 누락되었다고 표시됩니다.
~/.config/imap-smtp-email/.env을 검사하고 SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS 및 SMTP_FROM가 있는지 확인하도록 요청하세요.


먼저 전체 이메일을 표시하지 않고 초안을 보냅니다.
Claw에게 AGENTS.md을 검사하고 Outbound Email Protocols에서 전체 초안을 표시하고 모든 전송 전에 승인을 기다려야 하는지 확인하도록 요청하세요.


이메일 본문에 리터럴 \\n, 따옴표 또는 기타 이상한 문자가 표시됩니다.
Claw에 follow-up-email을 업데이트하여 승인 전과 전송 전에 본문이 일반 일반 텍스트로 렌더링되도록 요청하세요. 


테스트 이메일이 도착하지 않습니다.
먼저 스팸메일을 확인하세요. 


새 스킬이 활성화되지 않은 것 같습니다.
OpenClaw에 /new을 입력한 후 follow-up-email을 다시 테스트하세요.
