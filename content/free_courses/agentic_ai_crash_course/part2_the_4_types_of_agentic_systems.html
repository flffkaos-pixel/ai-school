2부: 에이전트 시스템의 4가지 유형(및 언제 무엇을 사용해야 하는지)

안녕하세요,

이전 부분에서는 AI 에이전트를 콘텐츠를 이해하거나 생성하는 것뿐만 아니라 작업을 수행하고 작업을 엔드 투 엔드로 처리하는 것에 대해서도 살펴보았습니다.

그러나 팀이 자신의 스택에 "에이전트를 추가"하려고 서두르면서 문제는 다음과 같습니다.
모든 에이전트가 동일하게 구축되는 것은 아니며 모든 문제에 고도로 자율적인 시스템이 필요한 것도 아닙니다.

이번 강의에서는 간단하지만 강력한 렌즈를 사용하여 네 가지 유형의 에이전트 시스템(어제 논의한 대로)을 살펴보겠습니다.

  • 에이전트는 얼마나 많은 자율성을 갖고 있나요?
  • 인간이나 시스템은 얼마나 많은 통제력을 유지합니까?

이러한 균형은 시스템 작동 방식, 평가 방식, 구축해야 하는 인프라에 영향을 미칩니다.

도구를 활용한 LLM

대부분의 최신 에이전트의 핵심에는 시스템의 두뇌 역할을 하는 LLM(대형 언어 모델)이 있습니다.
이 과정 전체에서 LLM이라는 용어를 사용하여 텍스트 전용 모델뿐만 아니라 생성 AI 모델을 광범위하게 지칭합니다.

자체적으로 콘텐츠를 생성할 수 있지만 이를 에이전트로 전환하려면 다음을 사용하여 콘텐츠를 보강해야 합니다.

  • 도구 ??호출할 수 있는 API, 함수, 데이터베이스
  • 계획 ??목표를 여러 단계로 나누는 능력
  • 메모리 ??과거의 행동과 결과를 추적할 수 있도록 합니다.
  • 상태 및 제어 논리 ??무엇을 알기 위해?셲 완료, 실패, 다음에 수행할 작업

이러한 구성 요소에 연결되면 LLM은 단순한 챗봇 이상의 역할을 하게 됩니다.
추론하고, 조치를 취하고, 적응할 수 있는 목표 중심 시스템이 됩니다.

하지만 감독 없이 행동하는 것을 얼마나 신뢰하느냐에 따라 결국 다양한 유형의 에이전트가 탄생하게 됩니다.
가장 자율적인 것부터 시작하여 살펴보겠습니다.

1. 규칙 기반 시스템/에이전트

낮은 자율성, 낮은 통제

이들 시스템은 LLM을 전혀 사용하지 않습니다.

잠깐만요?셳 AI 에이전트에 대해 이야기하고 있는 건가요?
그렇습니다.

어떤 문제를 해결하나요?
입력과 출력이 고정되어 있는 잘 구조화되고 반복적인 작업입니다.

예:

  • 고정 금액 이하의 환급을 자동으로 승인
  • 파일 이름 패턴을 기반으로 폴더의 파일 이름 바꾸기
  • Excel 시트의 데이터를 양식 필드에 복사

장점: 빠르고 감사 가능하며 예측 가능
단점: 변경하기 쉬우며 모호함을 처리할 수 있습니까?셳 모호함을 처리할 수 있습니까?
가장 적합한 경우: 모든 조건을 미리 알고 있고 유연성이 필요하지 않습니다.

2. 워크플로 에이전트

낮은 자율성, 높은 통제

이는 기업이 LLM을 워크플로에 도입하는 첫 번째 단계인 경우가 많습니다.
여기서 LLM은 기존 워크플로를 향상시키지만 작업을 독립적으로 실행하지는 않습니다.

어떤 문제를 해결하나요?
자연어 이해, 요약 또는 생성의 이점을 누리지만 여전히 인간의 의사결정이 필요한 반복적인 작업입니다.

예:

  • Zendesk와 같은 지원 도구에서 초안 응답 제안
  • 회의 기록 요약 생성
  • 자연어 쿼리를 BI 대시보드용 구조화된 검색 입력으로 변환

LLM 사용 방법:
입력(텍스트, 티켓, 문서)을 읽고, 맥락을 이해하고, 유용한 콘텐츠를 생성하지만 그에 따라 조치를 취하지는 않습니다.
무엇을 할지 결정하는 것은 여전히 ​​인간입니다.

장점: 손쉬운 배포, 낮은 위험, 빠른 가치
단점: 실행 또는 계획 가능, 제한된 엔드 투 엔드 가치
가장 적합한 경우: 팀을 강화하고 싶습니까?셲 감독을 포기하지 않고 생산성을 향상시키세요.

3. 반자동 에이전트

보통에서 높은 자율성, 보통의 통제

이것이 진정한 에이전트 시스템입니다.

어떤 문제를 해결하나요?
잘 이해되지만 인간에게는 너무 지루하거나 시간이 많이 걸리는 다단계 작업 흐름입니다.

예:

  • 결과를 기록하는 동시에 CRM 데이터를 기반으로 이메일 초안을 작성하고 개인화하고 전송하는 리드 후속 에이전트
  • 계약서 내용을 추출하고 내부 시스템을 업데이트하는 문서 자동화 에이전트
  • 여러 소스에서 데이터를 가져와 결과를 비교하고 구조화된 보고서를 보내는 연구 에이전트

LLM 사용 방법:
LLM은 단계를 계획하고, API를 호출하여 데이터를 가져오거나 푸시하고, 진행 상황을 추적하고, 문제가 발생하면 조정합니다.
여기에는 사람이 검토할 수 있는 대체 경로나 체크포인트가 포함되는 경우가 많습니다.

장점: 복잡한 워크플로 자동화, 시간 절약, 더 높은 ROI
단점: 인프라(계획, 메모리, 도구 호출)가 필요하고 테스트가 더 어렵습니다.
가장 적합한 경우: 일부 제어권을 유지하면서 잘 제한된 비즈니스 워크플로를 자동화하려는 경우.

4. 자율 에이전트

높은 자율성, 낮은 통제

이러한 에이전트는 완전히 목표 중심적입니다.

어떤 문제를 해결하나요?
여러 시스템이나 단계에 걸쳐 있으며 노력이 많이 들고, 비동기적이거나, 장기 실행되는 작업에는 지속적인 사람의 입력이 필요합니다.

예:

  • 며칠에 걸쳐 데이터를 수집하고, 업데이트를 요약하고, 주간 통찰력 브리핑을 생성하는 경쟁력 있는 조사 에이전트
  • 파이프라인의 문제를 감지하고, 근본 원인을 진단하고, 제안된 수정 사항이 포함된 티켓을 제출하는 운영 자동화 에이전트
  • 제품 흐름을 자율적으로 실행하고 결과를 기록하며 새로운 엣지 케이스 시나리오를 제안하는 테스트 에이전트

LLM 사용 방법:
LLM은 기획자, 의사 결정자, 도구 사용자, 메모리 추적자 및 의사소통자입니다.

장점: 확장성이 뛰어나고 복잡한 작업을 처리할 수 있음
단점: 모니터링하지 않으면 위험이 높으며 평가 또는 추적이 어렵고 인프라가 많음
가장 적합한 경우: 작업이 활용도가 높고 비동기적이며 모든 단계에서 사람의 피드백이 필요하지 않습니까?

무엇을 구축할지 결정하는 방법

좋아하는 아키텍처를 선택하는 것이 아닙니다.
문제부터 시작합니다.

스스로에게 물어보세요:

  • 반복적이고 구조적인가?
  • 언어 이해 또는 생성이 포함됩니까?
  • 의사결정이 필요한 다단계 작업인가요?
  • 전체 작업을 실행하기 위해 AI 시스템을 신뢰하십니까, 아니면 루프에 인간이 참여하기를 원하십니까?

여기?셲 핵심:

  • 이들 접근법은 상호 배타적이다.
  • 단일 시스템으로 이들을 혼합할 수 있습니다. 일부 부품은 높은 제어가 필요할 수 있고 다른 부품은 높은 자율성의 이점을 누릴 수 있습니다.
  • 각 문제 유형은 단일 에이전트 또는 협력 에이전트 그룹에 의해 처리될 수 있습니다.

과정 뒷부분에서 단일 에이전트와 다중 에이전트 설계에 대해 더 자세히 알아볼 것입니다.
지금은 다음 사항을 기억하세요.
