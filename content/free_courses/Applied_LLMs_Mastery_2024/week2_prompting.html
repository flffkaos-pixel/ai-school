[2주차] 프롬프트 및 프롬프트 엔지니어링

ETMI5: 5에서 설명해주세요

프롬프트 섹션에서는 원하는 출력을 생성하는 데 언어 모델을 안내하는 효과적인 프롬프트를 공식화하는 기본 사항을 배웁니다.

소개

격려

언어 모델 영역에서 "프롬프트"는 원하는 출력을 생성하기 위해 모델에 제공되는 정확한 지침이나 쿼리를 공식화하는 기술과 과학을 의미합니다.

신속한 엔지니어링

  • 빠르게 성장하는 분야인 프롬프트 엔지니어링은 다양한 응용 프로그램에서 언어 모델의 잠재력을 최대한 활용하기 위해 프롬프트를 개선하는 데 중점을 두고 있습니다.
  • 연구에서 프롬프트 엔지니어링은 질문 답변 및 산술 추론과 같은 작업 전반에 걸쳐 LLM의 성능을 향상시키는 강력한 도구입니다.
  • 프롬프트 작성 외에도 프롬프트 엔지니어링은 LLM과 상호 작용하고 개발하는 데 필수적인 풍부한 기술 세트입니다.
  • 이러한 숙련도는 AI 행동을 인간의 의도와 일치시키는 데 매우 중요합니다. 

                Image Source: [https://zapier.com/blog/prompt-engineering/](https://zapier.com/blog/prompt-engineering/)

프롬프트는 이러한 모델을 효과적으로 사용하는 데 중요한 측면입니다.

  1. 문맥 이해: LLM은 문맥을 이해하고 다양한 텍스트 데이터에서 학습된 패턴을 기반으로 응답을 생성하도록 교육을 받았습니다.
  2. 학습 데이터 패턴: 학습 중에 모델은 광범위한 텍스트로부터 학습하여 데이터에 존재하는 언어적 뉘앙스와 패턴을 포착합니다.
  3. 전이 학습: LLM은 전이 학습을 활용합니다.
  4. 문맥 응답을 위한 문맥 프롬프트: 모델이 훈련된 언어 및 맥락과 유사한 프롬프트를 사용함으로써 사용자는 유사한 맥락 내에서 콘텐츠를 이해하고 생성하는 모델의 능력을 활용할 수 있습니다.
  5. 편향 완화: 모델은 학습 데이터에 존재하는 편향을 상속받을 수 있습니다.

요약하자면, LLM 교육에는 대규모 데이터 세트로부터의 학습이 포함되며, 프롬프트는 사용자가 이러한 모델을 안내하여 유용하고 관련성이 높으며 정책을 준수하는 응답을 생성하는 수단입니다.

프롬프트 기본 사항

프롬프트의 기본 원칙에는 당면한 작업에 맞는 특정 요소를 포함하는 것이 포함됩니다.

  1. 지침: 모델이 수행하기를 원하는 작업이나 작업을 명확하게 지정하세요.
  2. 컨텍스트: 모델이 작업을 더 잘 이해하고 더 정확한 응답을 생성하는 데 도움이 되는 외부 정보 또는 추가 컨텍스트를 제공합니다.
  3. 입력 데이터: 답변을 원하는 입력 또는 질문을 포함합니다.
  4. 출력 표시기: 원하는 출력의 유형이나 형식을 정의합니다.

다음은 텍스트 분류 작업에 대한 예시 프롬프트입니다.

고급 프롬프트 기술


프롬프트 기법은 빠르게 발전하는 연구 영역을 구성하며, 연구자들은 최적의 성능을 위한 모델을 효과적으로 프롬프트하기 위한 새로운 방법을 지속적으로 탐구합니다. 




A. 단계별**단계적 모듈식 분해**


이러한 방법에는 복잡한 문제를 더 작고 관리 가능한 단계로 나누어 문제 해결을 위한 구조화된 접근 방식을 촉진하는 것이 포함됩니다.

아래 제공된 예에서 프롬프트는 주어진 그룹의 홀수의 합이 짝수인지 여부를 평가하는 것과 관련됩니다.

제로샷(Zero Shot)은 LLM이 체계적인 추론 과정을 거치도록 안내하기 위해 원래 질문에 "단계적으로 생각해보자"라는 메시지를 추가하는 것입니다.

Auto-CoT(Automatic Chain-of-Thought)는 시연을 위한 추론 체인 생성을 자동화하도록 설계되었습니다.

Auto-CoT 프로세스에는 두 가지 주요 단계가 포함됩니다.


    

  1. 질문 클러스터링: 유사성을 기준으로 질문을 클러스터로 분할합니다.
    2. 

  2. 데모 샘플링: 각 클러스터에서 대표적인 질문을 선택하고 간단한 경험적 방법으로 Zero-Shot-CoT를 사용하여 추론 체인을 생성합니다.
    3.

다음은 신속한 엔지니어링을 위한 잘 알려진 도구 모음입니다. 


    

  1. PromptAppGPT:
      


    • 기능: 로우 코드 프롬프트 기반 개발, GPT 텍스트 및 DALE 이미지 생성, 온라인 프롬프트 편집기/컴파일러/실행기, 자동 UI 생성, 플러그인 확장 지원.
      • 

    • 목표: GPT 기반의 자연어 앱 개발을 지원하여 GPT 애플리케이션 개발에 대한 장벽을 낮춥니다.
      • 

    

    2. 

  2. 프롬프트벤치:
      


    • 기능: 빠른 모델 성능 평가, 신속한 엔지니어링 방법(Few-shot Chain-of-Thought, 감정 프롬프트, 전문가 프롬프트), 적대적 프롬프트 평가, 잠재적인 테스트 데이터 오염을 완화하기 위한 동적 평가를 위한 사용자 친화적인 API입니다.
      • 

    • 목적: 프롬프트 엔지니어링 및 적대적 프롬프트 평가를 포함한 다양한 기능을 통해 LLM 평가를 촉진합니다.
      • 

    

    3. 

  3. 엔진:
      


    • 배경: GPT-3 및 Codex와 같은 LLM에 대한 신속한 엔지니어링을 단순화하는 것을 목표로 하며 모델의 특정 출력을 동축으로 만드는 입력 제작용 유틸리티를 제공합니다.
      • 

    • 목적: 프롬프트 생성 및 유지 관리를 촉진하고 프롬프트 엔지니어링과 관련된 패턴 및 관행을 체계화합니다.
      • 

    

    4. 

  4. AI의 적용:
      


    • 목표: 처음 GPT-3 사용자를 지원하고, 신속한 엔지니어링을 실험하고, 창의적인 글쓰기, 분류, 채팅 봇과 같은 사용 사례에 맞게 제품을 최적화합니다.
      • 

    

    5. 

  5. 오픈프롬프트:
      


    • 기능: 신속한 학습 파이프라인을 배포하고 허깅페이스 변환기에서 PLM 로드를 지원하기 위한 유연하고 확장 가능한 표준 프레임워크입니다.
      • 

    • 목표: 즉각적인 학습에 대한 표준화된 접근 방식을 제공하여 특정 NLP 작업에 대해 PLM을 더 쉽게 적용할 수 있도록 합니다.
      • 

    

    6. 

  6. 와:
      


    • 목표: LLM에 대한 신속한 테스트를 촉진하고, NLP 작업을 단순화하며, 프롬프트를 최적화하여 OpenAI 토큰 비용을 줄이는 것을 목표로 합니다.
      • 

    

    7.
