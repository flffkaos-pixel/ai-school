[1주차 2부] 도메인 및 작업 적응 방법

ETMI5: 5에서 설명해주세요

이 섹션에서는 전문 도메인에서 일반 AI 모델의 한계를 조사하고 도메인 적응 LLM의 중요성을 강조합니다.

도메인별 사전 훈련, 도메인별 미세 조정 및 RAG(검색 증강 생성)의 세 가지 유형의 도메인 적응 방법을 소개합니다.

LLM을 효과적으로 사용하기

ChatGPT와 같은 일반 AI 모델은 다양한 주제에 걸쳐 인상적인 텍스트 생성 능력을 보여주지만 특정 도메인에 필요한 깊이와 미묘한 이해가 부족할 수 있습니다.

도메인별 LLM을 사용하면 다음과 같은 이점이 있습니다.

  1. 심도와 정확성: 일반 LLM은 다양한 주제에 대한 텍스트 생성에 능숙하지만 전문 분야에 필요한 깊이와 뉘앙스가 부족할 수 있습니다.
  2. 한계 극복: 일반 LLM에는 잠재적인 부정확성, 맥락 부족, 환각에 대한 민감성 등의 한계가 있습니다.
  3. 향상된 사용자 경험: 분야별 LLM은 맞춤형 응답을 제공하여 향상된 사용자 경험에 기여합니다.
  4. 향상된 효율성 및 생산성: 기업은 분야별 LLM의 향상된 효율성을 통해 이익을 얻을 수 있습니다.
  5. 개인정보 보호 문제 해결: 의료 등 민감한 데이터를 다루는 업계에서 일반 LLM을 사용하면 개인정보 보호 문제가 발생할 수 있습니다.

이전 섹션을 떠올려 보면 특정 사용 사례에서 LLM을 사용하는 여러 가지 방법이 있었습니다.

  1. 제로샷 학습
  2. 퓨샷 학습
  3. 도메인 적응

제로샷 학습과 퓨샷 학습에는 예제를 통해 또는 관심 있는 특정 질문을 제시하여 일반 모델에 지시하는 작업이 포함됩니다.

BloombergGPT는 금융 산업 내 다양한 ​​작업에서 탁월한 성능을 발휘하도록 설계된 500억 개의 매개변수 언어 모델입니다.

  1. 금융 감정 분석: 뉴스 기사, 소셜 미디어 게시물, 재무 보고서 등 금융 텍스트의 감정을 분석하고 결정합니다.
  2. 지정 법인 인식: 금융 문서에 언급된 법인(예: 회사, 개인, 금융 상품)을 식별하고 분류합니다.
  3. 뉴스 분류: 금융 뉴스 기사를 다양한 주제나 클래스로 분류합니다.
  4. 금융 관련 질문 답변: 금융 주제와 관련된 질문에 답변합니다.
  5. 재무를 위한 대화 시스템: 금융과 관련된 자연어 대화에 참여합니다.

이를 달성하기 위해 BloombergGPT는 Bloomberg의 광범위한 아카이브에 있는 도메인별 금융 언어 문서를 공개 데이터세트와 결합한 대규모 데이터세트를 사용하여 도메인별 사전 학습을 거칩니다.

모델 아키텍처는 70개 레이어의 변환기 디코더 블록을 포함하는 이전 연구 노력의 지침을 기반으로 합니다(자세한 내용은 논문 참조).

도메인별 미세 조정

도메인별 미세 조정은 특정 작업이나 특정 도메인 내에서 기존 언어 모델을 개선하여 성능을 향상하고 해당 도메인의 고유한 상황에 맞게 조정하는 프로세스입니다.

?뮕 이전 방법, 즉 도메인별 사전 학습에는 특정 도메인의 데이터에만 언어 모델을 교육하여 해당 도메인에 대한 특수 모델을 만드는 것이 포함됩니다.

도메인별 미세 조정의 주요 단계는 다음과 같습니다.

  1. 사전 학습: 처음에는 대규모 언어 모델이 광범위한 데이터 세트에 대해 사전 학습되어 일반적인 언어 패턴, 문법 및 문맥 이해를 파악할 수 있습니다(일반 LLM).
  2. 세부 조정 데이터 세트: 원하는 도메인이나 작업에 맞춰 더욱 집중된 데이터 세트를 수집하거나 준비합니다.
  3. 미세 조정 프로세스: 사전 학습된 언어 모델은 이 도메인별 데이터 세트에 대한 추가 학습을 거칩니다.
  4. 작업 최적화: 미세 조정된 모델은 선택한 도메인 내의 특정 작업에 맞게 최적화됩니다.

도메인별 미세 조정은 다음과 같은 몇 가지 이점을 제공합니다.

  • 이를 통해 모델은 특정 도메인을 전문화하여 해당 도메인 내 작업의 효율성을 향상할 수 있습니다.
  • 사전 훈련 중에 얻은 지식을 활용하여 처음부터 모델을 훈련하는 것에 비해 시간과 계산 리소스를 절약합니다.
  • 모델은 대상 도메인의 특정 요구 사항과 미묘한 차이에 적응할 수 있어 도메인별 작업의 성능이 향상됩니다.

도메인별 미세 조정의 인기 있는 예는 온라인 의료 상담 플랫폼의 100,000개의 환자-의사 대화 데이터 세트를 사용하여 Meta-AI의 LLaMA(대형 언어 모델 메타 AI)에서 미세 조정된 전문 언어 모델인 ChatDoctor LLM입니다.

검색 증강 생성(RAG)

RAG(Retrieval Augmented Generation)는 생성 프로세스 중에 외부 소스로부터 상황에 맞는 최신 정보를 통합하여 LLM에서 생성된 응답의 품질을 향상시키는 AI 프레임워크입니다.

모델 업데이트(사전 훈련 및 미세 조정)를 위한 이전 접근 방식과 달리 RAG에는 특정한 장점과 단점이 있습니다.

RAG의 단점
정보 신선도: RAG는 외부 데이터베이스에서 최신 또는 상황별 데이터를 제공하여 LLM의 정적 특성을 해결합니다. 복잡한 구현(다중 이동 부분): RAG 구현에는 벡터 데이터베이스 생성, 모델 삽입, 검색 색인 등이 포함될 수 있습니다. RAG의 성능은 이러한 모든 구성 요소의 개별 성능에 따라 달라집니다.
도메인별 지식: RAG는 벡터 데이터베이스에서 관련 결과를 가져와 도메인별 지식으로 LLM을 보완합니다. 지연 시간 증가: RAG의 검색 단계에는 데이터베이스 검색이 포함되며, 이로 인해 외부 소스에 의존하지 않는 모델에 비해 응답 생성 시 지연 시간이 발생할 수 있습니다.
환각 및 인용 감소: RAG는 LLM을 검증 가능한 외부 사실로 근거화하여 환각 가능성을 줄이고 출처도 인용할 수 있습니다.
비용 효율성: RAG는 광범위한 모델 교육이나 미세 조정이 필요 없는 비용 효율적인 솔루션입니다.

RAG, 도메인별 미세 조정 및 도메인별 사전 학습 중에서 선택

다음 경우에 도메인별 사전 학습을 사용하세요:

  • 독점 도메인 초점: 사전 학습은 특정 도메인의 데이터에 대해서만 학습된 모델이 필요한 경우 해당 도메인에 대한 특수 언어 모델을 생성하는 데 적합합니다.
  • 모델 아키텍처 맞춤설정: 도메인의 특정 요구 사항에 따라 모델 아키텍처, 크기, 토크나이저 등의 다양한 측면을 맞춤설정할 수 있습니다.
  • 사용 가능한 광범위한 학습 데이터: 효과적인 사전 학습을 위해서는 모델이 선택한 영역의 복잡성을 포착하도록 하기 위해 많은 양의 영역별 학습 데이터가 필요한 경우가 많습니다.

도메인별 미세 조정을 사용하는 경우:

  • 전문화 필요: 사전 교육을 받은 LLM이 있고 이를 특정 작업이나 특정 도메인 내에서 조정하려는 경우 미세 조정이 적합합니다.
  • 작업 최적화: 선택한 도메인에서 최적의 성능을 위해 아키텍처, 크기, 토크나이저 등 작업과 관련된 모델의 매개변수를 조정할 수 있습니다.
  • 시간 및 리소스 효율성: 미세 조정은 사전 학습 단계에서 얻은 지식을 활용하므로 모델을 처음부터 학습하는 것에 비해 시간과 컴퓨팅 리소스를 절약합니다.

다음 경우에 RAG를 사용하세요:

  • 정보 최신성이 중요함: RAG는 외부 소스로부터 상황별 최신 데이터를 제공합니다.
  • 환각을 줄이는 것이 중요합니다. 검증 가능한 사실과 외부 지식 기반의 인용을 바탕으로 LLM을 구축하세요.
  • 비용 효율성이 최우선: 광범위한 모델 학습이나 미세 조정을 피하세요.
