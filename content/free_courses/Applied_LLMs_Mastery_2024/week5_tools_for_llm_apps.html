[5주차] LLM 지원서 작성을 위한 도구

ETMI5: 5에서 설명해주세요

본 과정의 이 섹션에서는 LLM 응용 프로그램의 생성 및 향상을 촉진하는 필수 기술과 도구를 살펴봅니다.

LLM 지원 유형

LLM 응용 프로그램은 점점 더 많은 신생 기업과 기업이 다양한 목적을 위해 LLM을 운영에 통합하면서 추진력을 얻고 있습니다.

  1. 맞춤 모델 조정: 이는 처음부터 맞춤 모델을 개발하는 것과 기존 모델을 미세 조정하는 것을 모두 포함합니다.
  2. RAG 기반 애플리케이션: 현재 가장 간단하고 널리 채택되는 접근 방식인 RAG(검색 증강 생성) 방법은 상황별 정보가 보완된 기본 모델을 활용합니다.

지난 몇 주 동안??내용에서는 이러한 방법론 간의 차이점을 다루고 특정 요구 사항에 따라 가장 적합한 방법을 선택하는 기준에 대해 논의했습니다.

다음 섹션에서는 이 두 가지 방법론 모두에 사용할 수 있는 도구 옵션을 살펴보겠습니다.

도구 유형

도구는 크게 네 가지 주요 그룹으로 분류할 수 있습니다.

  1. 입력 처리 도구: 애플리케이션에 대한 데이터와 다양한 입력을 수집하도록 설계된 도구입니다.
  2. LLM 개발 도구: 이 도구는 호출, 미세 조정, 실험 수행 및 조정을 포함하여 대규모 언어 모델과의 상호 작용을 용이하게 합니다.
  3. 출력 도구: 이 도구는 LLM 애플리케이션의 출력을 관리하는 데 사용되며 기본적으로 출력 후 프로세스에 중점을 둡니다.
  4. 애플리케이션 도구: 이 도구는 애플리케이션 호스팅, 모니터링 등 앞서 언급한 세 가지 구성요소의 포괄적인 관리를 감독합니다.

이전 콘텐츠에서 RAG가 어떻게 작동하는지 기억한다면 애플리케이션은 일반적으로 다음 단계를 따릅니다.

  1. 사용자로부터 쿼리를 받습니다(애플리케이션에 대한 사용자 입력).
  2. 임베딩 검색을 활용하여 관련 데이터를 찾습니다(여기에는 임베딩 LLM, 데이터 소스 및 데이터 임베딩을 저장하기 위한 벡터 데이터베이스가 포함됩니다).
  3. 처리를 위해 쿼리와 함께 검색된 문서를 LLM에 전달합니다.
  4. LLM의 출력을 사용자에게 다시 전달합니다.

아래 이미지에 설명된 것처럼 LLM 응답 호스팅 및 모니터링이 전체 애플리케이션 아키텍처에 통합되어 있습니다.

?뮕이러한 각 도구 범주가 왜 필요한지 아직도 확신하지 못한다면 지난 주의 내용을 검토하여 RAG 및 Fine-Tuning 응용 프로그램의 작동 방식을 이해하십시오.

LLM 앱 구축에 사용할 수 있는 도구 요약

입력 처리 도구

1. 데이터 파이프라인/소스

LLM 응용 프로그램에서는 데이터의 효과적인 관리 및 처리가 성능과 기능을 향상시키는 데 핵심입니다.

아.

  • 기존 ETL 도구: 확립된 ETL 솔루션은 데이터 워크플로를 관리하는 데 널리 사용됩니다.
  • 문서 로더 및 조정 프레임워크: 구조화되지 않은 데이터를 주로 처리하는 애플리케이션은 조정 프레임워크 내에 통합된 문서 로더를 활용하는 경우가 많습니다.
  • Llama Hub 생태계의 구성요소인 라마인덱스는 효율적인 데이터 관리를 위한 색인 생성 및 검색 기능을 제공합니다.

LlamaIndex 및 LangChain에 대한 자세한 내용은 오케스트레이션 섹션에서 제공됩니다.

베.

LLM 애플리케이션의 데이터 관리를 위한 기존 스택이 작동 중이지만, 특히 LLM 앱에 맞게 특별히 맞춤화된 데이터 복제 솔루션을 개발할 때 향상될 가능성이 있습니다.

구조화된 데이터와 구조화되지 않은 데이터를 위한 데이터 로더

다양한 소스의 데이터를 통합하는 기능은 구조화된 입력과 구조화되지 않은 입력을 모두 처리할 수 있는 데이터 로더를 통해 활성화됩니다.

  • 구조화되지 않은 데이터: Unstructured.io에서 제공하는 솔루션을 사용하면 복잡한 ETL 파이프라인을 생성할 수 있습니다.
  • 구조화된 데이터 소스: 데이터베이스 및 기타 구조화된 데이터 저장소에 직접 연결하는 로더가 사용되어 원활한 데이터 통합 ​​및 조작이 용이합니다.

2. 벡터 데이터베이스

RAG의 콘텐츠를 다시 참조하여 유사성 삽입을 통해 가장 관련성이 높은 문서를 식별하는 방법을 살펴보았습니다.

벡터 데이터베이스의 주요 역할은 임베딩(즉, 벡터)을 효율적으로 저장, 비교 및 ​​검색하는 것이며, 종종 수십억 개까지 확장됩니다.

벡터 데이터베이스의 스펙트럼은 다음을 포함하여 광범위합니다.

  • 위비에이트, 베스파, Qdrant와 같은 오픈소스 시스템: 이러한 플랫폼은 단일 노드 기반에서 탁월한 성능을 제공하고 특정 애플리케이션에 맞게 맞춤설정할 수 있으므로 맞춤형 플랫폼 개발에 대한 전문 지식을 갖춘 AI 팀 사이에서 선호되는 선택이 됩니다.
  • 크로마 및 파이스와 같은 로컬 벡터 관리 라이브러리: 뛰어난 개발자 경험으로 잘 알려진 이 라이브러리는 소규모 애플리케이션 및 개발 실험을 위해 구현하기가 쉽습니다.
  • pg벡터와 같은 OLTP 확장: 이 옵션은 다양한 데이터베이스 요구 사항에 대해 Postgres를 사용하는 경향이 있는 사용자나 대부분의 데이터 인프라를 단일 클라우드 공급자로부터 조달하여 벡터 지원을 위한 실행 가능한 솔루션을 제공하는 기업에 적합합니다.

기술이 발전함에 따라 많은 오픈 소스 벡터 데이터베이스 제공업체가 클라우드 서비스에 도전하고 있습니다.

LLM 개발 도구

1. 모델

개발자는 선택할 수 있는 다양한 모델 옵션을 갖고 있으며 각 모델 옵션은 프로젝트 요구 사항에 따라 고유한 장점을 갖고 있습니다.

애플리케이션이 개발에서 생산으로 이동함에 따라 비용과 성능의 균형을 맞추는 쪽으로 초점이 옮겨가는 경우가 많습니다.

독점 모델 외에도 오픈 소스 대안에 대한 관심이 높아지고 있으며 대부분은 포옹하는 얼굴에서 사용할 수 있습니다. 오픈 소스 모델은 유연하고 비용 효율적인 솔루션을 제공하며 특히 검색이나 채팅 기능과 같은 대용량 소비자 대면 애플리케이션에 유용합니다.

독점 모델과 오픈 소스 모델 사이의 선택은 비용에만 달려 있는 것이 아닙니다.

?뮕많은 독점 모델은 애플리케이션 개발자가 미세 조정할 수 없다는 점에 유의하세요.

2. 오케스트레이션

LLM 애플리케이션과 관련된 오케스트레이션 도구는 여러 구성 요소 및 LLM과의 상호 작용을 포함하는 복잡한 프로세스를 간소화하고 관리하도록 설계된 소프트웨어 프레임워크입니다.

  1. 프롬프트 엔지니어링 자동화: 조정 도구는 LLM에 전송되는 쿼리 또는 지침인 프롬프트의 생성 및 관리를 자동화합니다.
  2. 외부 데이터 통합: 외부 데이터를 프롬프트에 쉽게 통합하여 원래 학습되지 않은 컨텍스트로 모델의 응답을 향상합니다.
  3. API 상호작용 관리: 조정 도구는 모델 호출, API 키 관리, 모델에서 반환된 데이터 처리 등 LLM API와의 인터페이스의 복잡성을 처리합니다.
  4. 프롬프트 체인 및 메모리 관리: 하나의 LLM 상호 작용의 출력이 다른 LLM 상호 작용의 입력으로 사용되는 프롬프트 체인을 활성화하여 보다 정교한 대화 또는 데이터 처리 시퀀스를 허용합니다.
  5. 애플리케이션 개발 단순화: 조정 도구를 사용하면 LLM으로 직접 작업하는 복잡성을 추상화하여 개발자가 애플리케이션을 더 쉽게 구축할 수 있습니다.
  6. 공급업체 종속 방지: 이러한 도구는 모델에 구애받지 않도록 시스템을 설계하는 경우가 많습니다. 즉, 다양한 제공업체의 다양한 LLM과 협력할 수 있습니다.

LangChain 및 LlamaIndex와 같은 프레임워크는 프롬프트 체인, 외부 API와의 인터페이스, 벡터 데이터베이스의 상황별 데이터 통합, 여러 LLM 상호 작용에서 일관성 유지와 같은 복잡한 프로세스를 단순화함으로써 작동합니다.

이미지 출처: https://stackoverflow.com/questions/76990736/differences-between-langchain-llamaindex

프롬프트 내에 특정 데이터를 삽입하여 모델 출력을 개인화하는 검색 증강 생성 기술은 미세 조정을 통해 모델의 가중치를 변경하지 않고 개인화가 어떻게 달성될 수 있는지 보여줍니다.

언어 모델 API의 가용성은 강력한 모델에 대한 액세스를 민주화하여 전문 기계 학습 팀을 넘어 더 광범위한 개발자 커뮤니티로 사용을 확장합니다.

3. 컴퓨팅/훈련 프레임워크

컴퓨팅 및 교육 프레임워크는 LLM 애플리케이션의 개발 및 배포에 필수적인 역할을 하며, 특히 특정 요구 사항에 맞게 모델을 미세 조정하거나 완전히 새로운 모델을 개발할 때 더욱 그렇습니다.

컴퓨팅 프레임워크

컴퓨팅 프레임워크와 클라우드 서비스는 LLM 애플리케이션을 효율적으로 실행하는 데 필요한 확장 가능한 리소스를 제공합니다.

  • 클라우드 제공업체: AWS(Amazon 웹 서비스)와 같은 서비스는 LLM 애플리케이션의 훈련 및 추론 단계 모두에 중요한 GPU 및 CPU 인스턴스를 포함하여 광범위한 컴퓨팅 리소스를 제공합니다.
  • LLM 인프라 회사: Fireworks.ai 및 모든 규모과 같은 회사는 LLM에 맞춤화된 인프라 솔루션을 전문적으로 제공합니다.

교육 프레임워크

LLM의 개발 및 미세 조정을 위해 딥 러닝 프레임워크가 사용됩니다.

  • PyTorch: 유연성, 사용 용이성 및 동적 계산 그래프로 인해 LLM 교육을 위한 연구원 및 개발자 사이에서 인기 있는 선택입니다.
  • TensorFlow: LLM 교육 및 배포를 위한 강력한 지원을 제공하는 널리 사용되는 또 다른 프레임워크입니다.

?뮕RAG를 활용하는 것과 같은 LLM API 애플리케이션은 API를 통해 제공되는 사전 학습된 모델을 사용하기 때문에 일반적으로 학습을 위해 계산 리소스에 직접 액세스할 필요가 없습니다.

4. 실험 도구

실험 도구는 초매개변수, 미세 조정 기술 및 모델 자체의 탐색 및 최적화를 촉진하므로 LLM 응용 프로그램의 중추적인 역할을 합니다.

?뮕혻언급된 도구는 실험이 핵심인 모델의 미세 조정 또는 교육과 관련된 시나리오에 주로 유용하다는 점에 유의하는 것이 중요합니다.

아래는 몇 가지 실험 도구입니다.

  • 실험 추적: 플레이어 및 편향과 같은 도구는 시간 경과에 따른 초매개변수, 모델 아키텍처 및 성능 측정항목의 변화를 포함하여 실험을 추적하기 위한 플랫폼을 제공합니다.
  • 모델 개발 및 호스팅: Hugging Face 및 ML흐름와 같은 플랫폼은 맞춤 LLM을 포함하여 ML 모델을 개발, 공유, 배포하기 위한 생태계를 제공합니다.
  • 성능 평가: 통계와 같은 도구는 실제 프로덕션 환경에서 모델 성능을 평가하는 기능을 제공하므로 개발자는 A/B 테스트를 수행하고 모델 동작에 대한 실제 피드백을 수집할 수 있습니다.

응용 도구

1. 호스팅

오픈 소스 모델을 활용하는 개발자는 원하는 대로 다양한 호스팅 서비스를 이용할 수 있습니다.

LLM 구성 요소 외에도 LLM 응용 프로그램의 정적 요소(본질적으로 모델 자체를 제외한 모든 요소)에도 호스팅 솔루션이 필요합니다.

2. 모니터링

모니터링 및 관찰 도구는 특히 프로덕션 환경에 배포한 후 애플리케이션을 유지 관리하고 개선하는 데 필수적입니다.

이 분야에서 주목할만한 발전 중 하나는 WhyLabs의 랭킷 출시입니다.
