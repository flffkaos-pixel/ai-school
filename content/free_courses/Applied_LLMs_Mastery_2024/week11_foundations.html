[11주차] LLM 기초

ETMI5: 5에서 설명해주세요

과정의 첫 주에 우리는 두 가지 유형의 기계 학습 모델, 즉 LLM이 포함된 생성 모델과 차별 모델 간의 차이점을 살펴보았습니다.

생성 모델과 차별 모델

첫 주에는 Generative AI의 개념을 간략하게 다루었습니다.

생성 모델

생성 모델은 데이터가 생성되는 방식을 이해하려고 노력합니다.

예를 들어, 개 이미지에 대한 생성 모델이 있는 경우 개를 구성하는 특징(예: 털, 귀, 꼬리)을 학습한 다음 이전에 한 번도 본 적이 없더라도 사실적으로 보이는 새로운 개 이미지를 생성할 수 있습니다.

차별 모델

반면, 판별 모델은 받은 입력을 기반으로 결정이나 예측을 내리는 데 중점을 둡니다.

개 이미지의 동일한 예를 사용하여 판별 모델은 이미지를 보고 개가 포함되어 있는지 여부를 결정합니다.

따라서 생성 모델은 데이터의 기본 패턴을 학습하여 새로운 샘플을 생성하는 반면, 판별 모델은 데이터가 생성된 방식에 대해 걱정하지 않고 입력 데이터를 기반으로 결정이나 예측을 내리는 데 중점을 둡니다.

기본적으로 생성 모델은 생성하는 반면 차별 모델은 분류하거나 예측합니다.

언어를 위한 신경망

수년 동안 신경망은 기계 학습의 핵심이었습니다.

순환 신경망(RNN)

순환 신경망(RNN)은 네트워크 아키텍처 내의 루프를 통해 정보가 지속되도록 하여 순차 데이터를 처리하도록 설계된 인공 신경망의 한 유형입니다.

RNN의 기본 원칙은 네트워크의 한 단계에서 다음 단계로 정보가 전달될 수 있도록 하는 방향성 주기를 형성하는 연결이 있다는 것입니다.

이미지 출처: https://colah.github.io/posts/2015-08-Understanding-LSTMs/

다음은 RNN 작동 방식에 대한 간단한 설명입니다.

  1. 입력 처리: 각 시간 단계 $t$에서 RNN은 $x_t$ 입력을 받습니다.
  2. 상태 업데이트: 입력 $x_t$는 이전 시간 단계의 네트워크 내부 상태 $h_{t-1}$와 결합되어 네트워크 내 가중치 연결(매개변수) 세트를 사용하여 새로운 상태 $h_t$를 생성합니다.
  3. 출력 생성: 현재 상태 $h_t$는 현재 시간 단계에서 출력 $y_t$을 생성하는 데 사용됩니다.
  4. 반복 연결: RNN의 주요 기능은 시간이 지남에 따라 네트워크를 통해 정보가 흐를 수 있도록 하는 반복 연결이 있다는 것입니다.

RNN은 순차 데이터를 처리하기 위한 강력한 모델이지만 장거리 종속성을 학습하는 데 어려움이 있고 훈련 중 그라데이션 소멸/폭발 문제와 같은 특정 제한 사항이 있을 수 있습니다.

장단기 기억(LSTM)

따라서 LSTM 네트워크는 RNN과 마찬가지로 텍스트와 같은 데이터 시퀀스를 더 잘 처리하도록 설계된 RNN의 향상된 버전이지만 다음과 같은 개선 사항이 있습니다.

이미지 출처: https://colah.github.io/posts/2015-08-Understanding-LSTMs/

  1. 메모리 셀: LSTM에는 시간이 지남에 따라 정보를 저장할 수 있는 특수 메모리 셀이 있습니다.
  2. 게이팅 메커니즘: LSTM은 게이트를 사용하여 메모리 셀 안팎으로의 정보 흐름을 제어합니다.
    • Forget Gate: 오래된 정보를 얼마나 잊어버릴지 결정합니다.
    • 출력 게이트: 현재 셀 상태를 얼마나 출력할지 결정합니다.
  3. 경사 흐름: LSTM은 훈련 중에 경사 흐름을 더 좋게 하여 긴 데이터 시퀀스에서 학습하는 데 도움이 됩니다.
  4. 장기 종속성 학습: LSTM은 시퀀스 초기의 중요한 정보를 기억하는 데 능숙하므로 장거리 상황에 대한 이해가 중요한 작업에 유용합니다.

따라서 LSTM은 중요한 정보를 기억하고 필요하지 않은 것을 잊어버려 시퀀스를 더 잘 처리하므로 언어 ​​처리와 같은 작업에서 기존 RNN보다 더 효과적입니다.

RNN과 LSTM(및 그 변형)은 모두 단어 시퀀스에서 다음 단어를 예측하는 것이 목표인 언어 모델링 작업에 널리 사용됩니다.

시퀀스 간(Seq2Seq) 모델

이것이 바로 Sequence-to-Sequence(Seq2Seq) 모델이 등장하는 곳입니다.

우리가 이겼나요? 간략하게 설명하기 위해 여기서 작업에 너무 깊이 빠져들었습니다.

이미지 출처: https://towardsdatascience.com/sequence-to-sequence-model-introduction-and-concepts-44d9b41cd42d

Seq2Seq 모델 + 주의

이미지 출처: https://lena-voita.github.io/nlp_course/seq2seq_and_attention.html

기존 Seq2Seq 모델의 문제점은 특히 가변 길이의 출력 시퀀스를 생성할 때 긴 입력 시퀀스를 효과적으로 처리할 수 없다는 것입니다.

이러한 문제를 해결하기 위해 주의 메커니즘이 도입되었습니다.

주의가 작동하는 방식은 다음과 같습니다.

  1. 인코더 표현: 먼저 입력 시퀀스가 ​​인코더에 의해 처리됩니다.
  2. 주의 가중치 계산: 디코딩 중에 디코더는 입력 시퀀스의 어느 부분에 집중할지 결정해야 합니다.
  3. 소프트맥스 정규화: 주의 가중치를 계산한 후 모델은 소프트맥스 함수를 사용하여 이를 정규화합니다.
  4. 가중 합계: 주의 가중치를 계산하고 정규화한 다음 모델은 인코더 숨겨진 상태의 가중 합계를 취합니다.
  5. 컨텍스트와 디코더 상태 결합: 마지막으로 가중치 합계에서 얻은 컨텍스트 벡터가 디코더의 현재 상태와 결합됩니다.
  6. 각 디코딩 단계마다 반복: 시퀀스 끝 토큰이 생성되거나 최대 길이에 도달할 때까지 각 디코딩 단계마다 2~5단계가 반복됩니다.

변압기 모델

주의가 필요한 Seq2Seq 모델의 문제점은 계산 비효율성과 긴 시퀀스에서 종속성을 효과적으로 캡처할 수 없다는 점에 있습니다.

Transformer 모델은 이러한 제한 사항을 해결하고 시퀀스 간 작업의 효율성과 효과를 개선하기 위해 도입되었습니다.

이미지 출처: https://arxiv.org/pdf/1706.03762.pdf

  1. Self-Attention 메커니즘: 인코더와 디코더 사이의 Attention 메커니즘에만 의존하는 대신 Transformer 모델은 self-attention 메커니즘을 도입합니다.
  2. 병렬화: Transformer 모델은 입력 시퀀스의 각 위치에 대해 병렬로 계산할 수 있는 self-attention 레이어를 사용합니다.
  3. 위치 인코딩: Transformer 모델은 순환 레이어를 사용하지 않으므로 입력 시퀀스의 요소 순서에 대한 고유 정보가 부족합니다.
  4. Transformer 아키텍처: Transformer 모델은 기존 Seq2Seq 모델과 유사한 인코더-디코더 아키텍처로 구성됩니다.

요약하면 Transformer 모델은 self-attention 메커니즘, 병렬화, 위치 인코딩 및 유연한 아키텍처를 도입하여 Seq2Seq 모델의 한계를 주의 깊게 해결합니다.

이전 언어 모델

LLM은 최근 특히 OpenAI의 GPT와 같은 모델에서 큰 주목을 받았지만 이 아키텍처의 토대가 아래 설명된 BERT, GPT(이전 버전) 및 T5와 같은 이전 모델에 의해 마련되었다는 점을 인식하는 것이 중요합니다.

BERT(변환기의 양방향 인코더 표현), GPT(생성 사전 훈련된 변환기) 및 T5(텍스트-텍스트 전송 변환기)와 같은 LLM은 다음 단계를 사용하여 변환기 모델(이전 섹션에서 설명)에 의해 도입된 개념을 기반으로 구축됩니다.

  1. 사전 학습 및 미세 조정: 이 모델은 사전 학습 및 미세 조정 접근 방식을 활용합니다.
  2. 양방향 컨텍스트: BERT는 마스크된 언어 모델링 목표를 활용하여 양방향 컨텍스트 모델링을 도입했습니다.
  3. 자동 회귀 생성: GPT 모델은 자동 회귀 생성을 활용합니다. 여기서 모델은 이전에 생성된 토큰을 기반으로 순서대로 다음 토큰을 예측합니다.
  4. 텍스트-텍스트 접근 방식: T5는 모든 NLP 작업이 텍스트-텍스트 매핑 문제로 구성되는 통합된 텍스트-텍스트 프레임워크를 도입합니다.
  5. 대규모 훈련: 이 모델은 수십억 개의 토큰이 포함된 대규모 데이터 세트에서 대규모 계산 리소스와 분산 훈련 기술을 활용하여 훈련됩니다.

대규모 언어 모델

Llama 및 ChatGPT와 같은 최신 Llama는 몇 가지 주요 방식으로 BERT 및 GPT와 같은 이전 모델에 비해 상당한 발전을 나타냅니다.

  1. 작업 전문화: BERT 및 GPT와 같은 이전 LLM은 텍스트 분류, 언어 생성, 질문 응답을 포함한 광범위한 NLP 작업을 수행하도록 설계되었지만 Llama 및 ChatGPT와 같은 최신 모델은 더욱 전문화되었습니다.
  2. 다중 모드 기능: Llama 및 기타 최신 LLM은 다중 모드 기능을 통합하여 이미지, 오디오, 비디오 등 다른 형식과 함께 텍스트를 처리하고 생성할 수 있습니다.
  3. 효율성 향상: 최근 LLM 아키텍처 및 교육 방법론의 발전으로 효율성이 향상되어 Llama 및 ChatGPT와 같은 모델이 더 적은 매개변수와 계산 리소스를 사용하여 이전 모델과 비슷한 성능을 달성할 수 있게 되었습니다.
  4. 미세 조정 및 전이 학습: ChatGPT와 같은 LLM은 특정 데이터세트나 작업을 미세 조정하여 대상 도메인의 성능을 더욱 향상시키는 경우가 많습니다.
  5. 대화형 및 동적 응답: ChatGPT 및 유사한 대화 모델은 자연어 대화에서 대화형 및 동적 응답을 생성하도록 설계되었습니다.
